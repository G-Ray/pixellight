Installation


General

It is highly recommended to use the provided installer to install the PixelLight SDK on your system. The installer will add the PixelLight 'Runtime' path to the ’PATH’ environment variable of Windows. This is required to enable Windows to find the different PixelLight dll’s and therefore to use the PixelLight SDK in the proper way. If the PixelLight runtime is registered, it is not required to copy the PixelLight dll’s into each of your project directories - and therefore you can ensure that ALL of your projects in development will always use the same and up to date PixelLight version. This will avoid an unproductive version chaos.

The installer also adds a key to the registry, so that the path to the installed SDK can be found during runtime. This key is normally found at 'HKEY_LOCAL_MACHINE\SOFTWARE\PixelLight\PixelLight-SDK\Runtime' (or within 'HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\PixelLight\PixelLight-SDK\Runtime' if you are using a 32 bit PixelLight SDK on a 64 bit Windows)

Note that it is IMPORTANT that there’s only ONE installed PixelLight SDK! If there are multiple PixelLight SDK’s, there will be conflicts because Windows can’t know which of the 3 for instance PLCore.dll versions is the correct one for your task.



PixelLight runtime directory

PixelLight requires some standard data that you will find in the PixelLight SDK directory 'Runtime'. Within the package 'Runtime/Data/Standard.zip' are all default textures, texts etc. that are required by the framework. Within the 'Runtime'-directory itself you will find all dll’s which a PixelLight program will need to run. Note that all applications in the PixelLight SDK will use the data directly from this directory and therefore the data and dll’s are not duplicated, which will ensure that you always use the current versions. If you want to publish your application, you have to copy this data into your project directory to ensure that the user will have all required files! Set '*application name*.cfg -> UsePixelLightRuntime' you can find within your project executable directory to '0' to ensure that there are no conflicts with a PixelLight SDK installed on another system.

Not ALL dll’s need to be copied into your release package. For instance do NOT publish the debug versions (normally with ’D’ at the end of the name like 'PLMathD.dll'). Only release the release versions. Note that you should not copy unused files into your release project directory - that would only be a waste of resources. If you don’t need certain plugins/backends you can also ignore them during coping.

Examples:
- If no 'Null'-renderer is required ignore: PLRendererNull.plugin and PLRendererNull.dll
- If no 'ODE'-physics plugin is required ignore: PLPhysicsODE.plugin, PLPhysicsODE.dll and ode.dll

As you see, PixelLight is extremely modular and various provided components can be ignored. Each PixelLight plugin comes with a 'plugin'-file. Within this file you can for example see the external references of the plugin.

Do NOT force the end user of your application to install a PixelLight SDK in order be able to run your application. Do always make a release that contains everything the user will need to be able to run your application out of the box without the need to install further software!



Windows path variable

The PixelLight SDK will automatically add the 'Runtime'-path to your Windows path variable if you have installed it correctly. That is required if the PixelLight dll’s are not in the same directory as the executables! Therefore Windows will know where to look for the required dlls.

You are also able to edit this Windows path variable by hand:
1. Right click on the ’My Computer’ icon on the desktop.
2. Click on the ’Advanced’ tab.
3. Click the ’Environment Variables’ Button
4. Under the ’User Variables’ header, select the PATH entry and click ’Edit’
5. Scroll to the end of the text, add a new semi-colon and enter the location of your PixelLight runtime directory. (for example 'c:\PixelLight-SDK\Runtime\')
6. Click ’Ok’ on all the panels to exit this dialog.

After the path variable of Windows was manipulated, you may need to restart our system! Whether or not a system restart is requires depends on various factors like OS version.


Microsoft Visual Studio

You have to give your development environment the correct directories where to find the PixelLight relevant data: In your programming development environment like Microsoft Visual Studio you have to setup these things:
  • Include files: Here you have to add the PixelLight include directory ’<YourSDKPath>/Include/’ to enable the tool to find the headers etc. automatically. (for instance if you include #include <PLCore/String/String.h>)
  • Library files: Where to find the libraries. Like the dll’s itself all libraries are all in one directory ’<YourSDKPath>/Lib/’. In Microsoft Visual Studio 2010 you will find this global options under: Menu -> Tools -> Options -> Projects -> VC++ directories. You can also set this paths per project.
