Introduction



This documentation, focused on the Windows platform, will give you an overview over the PixelLight SDK. For more detailed information about the various parts have a look at the different documentations located in the ’Docs’-directory.

The PixelLight API is a strictly object-oriented API which offers efficient interfaces to the different functions. Therefore it is useful to have experience in object orient programming (OOP) to understand all used techniques and be able to use its full power... but you can also create simple programs if you are new to OOP or even programming in general - in this case the documentation and the example programs will show you how to develop your own first PixelLight project step by step. Normally, there is no need for low level access in order to develop a project, but it is still possible to do various things yourself. Therefore the PixelLight API is extremely flexible, as it offers high level functionality for efficient production, but still lets you do to things yourself to avoid being limited by the framework's functionality. Furthermore, the framework is easily expandable through its unique and flexible runtime type information system (RTTI). It is strongly recommended that you don't modify the framework source code and headers, because this can lead to problems if there are new framework versions. Instead, you should contact us in order to work out whether an implementation is possible in a certain time frame. Moreover, it would be nice if you inform us if you find some bugs or mistakes within the framework documentation - in most cases these problems can be solved quite easily.


SDK directories

  • Docs: PixelLight documentations
  • Include: The C++ header files (see section ’Installation’)
  • Lib: The C++ library files (see section ’Installation’)
  • Runtime: SDK runtime data (see section ’Installation’)
  • Samples: Some sample applications - as programmer your journey starts here
  • Tools: In here, you'll find the PixelLight viewer and Autodesk 3ds Max exporters

SDK version number

This is how the version number of the SDK is constructed:

PixelLight-SDK-< Major>.< Minor>.< Patch>-< Release>-< Platform>.< Extension>

- 'Major' is the number of the SDK main release for reached major milestones
- 'Minor' for reached minor milestones
- 'Patch' multiple bugfixes or medium size changes
- 'Release' is mainly for minor bug fixes or changes
- 'Platform' Platform the SDK is for, for example 'Windows' for a Windows version of the SDK
- 'Extension' is the file extension, for example 'exe' for a Windows version of the SDK

We strongly recommend to NOT use 'nightly build'-versions within a productive environment because they may be instable. This, usually automatically, build versions have a 'nightly' within the SDK name.
